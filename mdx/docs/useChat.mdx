# useChat()

## What is useChat?

The <KBD>useChat</KBD> hook is one of the core utilities provided by the Vercel AI SDK for building conversational AI applications.
In this section, we'll focus on covering the basics of message handling, API communication, streaming, and state management.
The Vercel AI SDK offers many more advanced capabilities, so for a deeper dive into the hook and its full feature set,
we recommend exploring the [Official Vercel AI SDK documentation](https://ai-sdk.dev/docs/reference/ai-sdk-ui/use-chat).

The useChat hook returns a set of functions & states that can be used to build AI interfaces fast such as:

- [sendMessage](#sendmessage)
- [messages](#messages)
- [status](#status)
- [error](#error)
- [regenerate](#regenerate)
- [stop](#stop)

## sendMessage

<KBD>sendMessage</KBD> is a function provided by the <KBD>useChat</KBD> hook
that sends a user's message to the configured API route and automatically
updates the conversation state.

When called, it handles the entire request lifecycle, including sending the message,
receiving the model's response, updating the messages array, and managing loading and error states.

This eliminates the need to manually make API requests or manage chat state,
allowing developers to focus on building the user experience rather than handling communication between the client and the AI model.

## messages

<KBD>messages</KBD> is an array provided by the <KBD>useChat</KBD> hook that
contains the complete conversation between the user and the AI model. Every
message sent by the user and every response generated by the model is
automatically added to this array.

The messages array serves as the source of truth for the chat interface,
making it easy to render conversations in the UI while also providing the context
needed for the model to understand previous interactions. As new messages are exchanged,
the array is automatically updated by the Vercel AI SDK.

The messages array contains objects of type <KBD>UIMessage</KBD>

## status

<KBD>status</KBD> is a value provided by the <KBD>useChat</KBD> hook that
represents the current state of a chat request. It can be used to provide
real-time feedback to users, such as showing loading indicators , disabling
inputs while a response is being generated, or displaying error states.

The status has 4 states that indicate different information about a request:

- **submitted** : Request has been sent
- **streaming** : Receiving the response. When using the streamText this becomes true when the response is being typed out.
- **error** : Indicates and error prevented the message from being sent.
- **ready** : Shows an idle state

## error

The hook returns an <KBD>error</KBD> object that captures anything that goes
wrong during a request such as network failures, invalid API keys, rate limits, or
model errors.

<KBD>error</KBD> is either an <KBD>Error</KBD> object or <KBD>undefined</KBD>.

## regenerate

The <KBD>regenerate</KBD> function allows you to re-run the AI response for a specific
message. This is useful when the user is not happy with the response and wants
a fresh one without having to retype their message.

## stop

The <KBD>stop</KBD> function cancels the current AI response mid-stream. This is useful
when the response is taking too long or the user decides they no longer need it.
